# 服务器端对接文档

## 1. 通信方式

TCP 长连接，服务端监听端口，我方（ELF2 开发板）主动连接。

| 项目 | 内容 |
|------|------|
| **协议** | TCP |
| **服务端监听地址** | `0.0.0.0:5566` |
| **我方连接地址** | `localhost:5566`（通过 SSH 隧道转发） |
| **字节序** | 大端 (Big-Endian) |
| **保活机制** | 心跳帧 + 空闲检测自动重连 |

## 2. 二进制帧结构

```
┌──────────┬──────────┬──────────┬──────────┐
│ Header (17B)        │ Payload  │ CRC32 (4B) │
│                      │ (N bytes)│             │
└──────────────────────┴──────────┴─────────────┘
```

### 帧头 (Header) — 固定 17 字节

| 偏移 | 字段 | 类型 | 字节数 | 说明 |
|------|------|------|--------|------|
| 0 | Sync Magic | uint16 | 2 | 同步魔数 `0xAA55`，用于帧边界检测 |
| 2 | Frame ID | uint16 | 2 | 帧序号 (0~65535，溢出回绕) |
| 4 | Message Type | uint8 | 1 | 消息类型码 (见下表) |
| 5 | Payload Length | uint32 | 4 | 有效载荷长度 (字节数) |
| 9 | Timestamp | uint64 | 8 | 发送时刻 Unix 毫秒时间戳 (NTP 同步) |

### 帧尾 (Footer) — 4 字节

| 偏移 | 字段 | 类型 | 字节数 | 说明 |
|------|------|------|--------|------|
| 0 | CRC32 | uint32 | 4 | 对 Header + Payload 整体计算的 CRC32 校验值 |

## 3. 消息类型

### 3.1 心跳 `0x00`

| 项目 | 内容 |
|------|------|
| **Type Code** | `0x00` |
| **Payload 长度** | 0 |
| **帧总长** | 21 字节 (17+0+4) |
| **发送时机** | 空闲超过 1.5 秒时自动发送 |

```
0xAA55 [frame_id] 0x00 0x00000000 [timestamp_ms:8] [CRC32:4]
```

### 3.2 注意力 `0x01`

| 项目 | 内容 |
|------|------|
| **Type Code** | `0x01` |
| **Payload 长度** | 4 (1 个 float32) |
| **帧总长** | 25 字节 |

```
0xAA55 [frame_id] 0x01 0x00000004 [timestamp_ms:8] [attention:f32] [CRC32:4]
```

| 字段 | 类型 | 说明 |
|------|------|------|
| attention | float32 | 注意力系数 0.0~1.0（0.0=专注, 1.0=放松） |

### 3.3 机器人速度 `0x02`

| 项目 | 内容 |
|------|------|
| **Type Code** | `0x02` |
| **Payload 长度** | 8 (2 个 float32) |
| **帧总长** | 29 字节 |

```
0xAA55 [frame_id] 0x02 0x00000008 [timestamp_ms:8] [linear:f32] [angular:f32] [CRC32:4]
```

| 字段 | 类型 | 说明 |
|------|------|------|
| linear | float32 | 线速度 (m/s)，对应 ROS2 `Twist.linear.x` |
| angular | float32 | 角速度 (rad/s)，对应 ROS2 `Twist.angular.z` |

### 3.4 情绪 `0x03`

| 项目 | 内容 |
|------|------|
| **Type Code** | `0x03` |
| **Payload 长度** | 12 (3 个 float32) |
| **帧总长** | 33 字节 |

```
0xAA55 [frame_id] 0x03 0x0000000C [timestamp_ms:8] [positive:f32] [neutral:f32] [negative:f32] [CRC32:4]
```

| 字段 | 类型 | 说明 |
|------|------|------|
| positive | float32 | 积极情绪概率 |
| neutral | float32 | 中性情绪概率 |
| negative | float32 | 消极情绪概率 |

**注意：** 三值之和应 ≈ 1.0（允许浮点误差）。若偏差过大（如 >0.1），建议丢弃该帧。

### 3.5 脑电波形 `0x04`

| 项目 | 内容 |
|------|------|
| **Type Code** | `0x04` |
| **Payload 长度** | `3 + channels × samples × 4` 字节 |
| **帧总长** | `24 + channels × samples × 4` 字节 |

```
0xAA55 [frame_id] 0x04 [payload_len:4] [timestamp_ms:8]
[channels:u8] [samples:u16] [data:int32[channels × samples]] [CRC32:4]
```

| 字段 | 类型 | 说明 |
|------|------|------|
| channels | uint8 | 通道数 (当前固定 8) |
| samples | uint16 | 每通道采样点数 |
| data | int32[] | 脑电原始数据，交错排列 (C1S1, C2S1, ..., C1S2, C2S2, ...) |

## 4. 连接与重连规则

1. 我方作为 TCP 客户端，连接服务端 `5566` 端口
2. 正常情况下通过 SSH 隧道（`ssh -L 5566:localhost:5566`）连接云端
3. 断线后每 1 秒自动重连，最多缓存 200 帧待发送
4. 重连成功后优先发送缓存帧，再发送实时数据
5. 服务端收到帧后应：
   - 通过 `0xAA55` 定位帧边界
   - 验证 CRC32，丢弃校验失败的帧
   - 记录已成功处理的帧 ID，检测丢帧

## 5. 服务端数据解析示例 (Python)

```python
import struct, zlib

SYNC = 0xAA55

def parse_frame(data):
    # 查找同步头
    idx = data.find(struct.pack('>H', SYNC))
    if idx < 0:
        return None

    header = struct.unpack('>HHB I Q', data[idx:idx+17])
    magic, frame_id, msg_type, payload_len, ts = header
    if magic != SYNC:
        return None

    payload = data[idx+17:idx+17+payload_len]
    frame_body = data[idx:idx+17+payload_len]
    crc_received = struct.unpack('>I', data[idx+17+payload_len:idx+21+payload_len])[0]
    crc_calc = zlib.crc32(frame_body) & 0xFFFFFFFF
    if crc_received != crc_calc:
        return None  # CRC 校验失败

    return msg_type, frame_id, ts, payload
```
